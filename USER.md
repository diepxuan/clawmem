Tài liệu này ghi lại kỳ vọng ổn định dành cho AI agent làm việc với Sếp trong repository này.

## Kỳ vọng khi làm việc

Sếp yêu cầu:

- Tuân thủ Git nghiêm ngặt.
- Làm theo quy trình nhánh → commit → PR.
- Không merge nếu chưa được lệnh rõ.
- Không force-push nếu chưa được lệnh rõ.
- Không commit trạng thái cục bộ hoặc bí mật.
- Cập nhật tài liệu cùng với thay đổi code, cấu hình, hoặc quy trình.
- Báo cáo kiểm định rõ ràng, bao gồm cả phần chưa chạy được và lý do.

## Định dạng báo cáo khi xong việc

Báo cáo nên có:

- Nhánh.
- SHA commit và thông điệp commit.
- Liên kết PR nếu có tạo hoặc cập nhật PR.
- File đã thay đổi.
- Kiểm định đã chạy.
- Trạng thái PR hoặc kiểm tra nếu có.
- Rủi ro hoặc file chưa theo dõi còn lại.

## Ranh giới quyết định

Hỏi Sếp trước khi:

- Mở rộng phạm vi tác vụ.
- Tạo PR ngoài yêu cầu.
- Đổi chiến lược phát hành hoặc phiên bản.
- Xóa file hoặc thư mục lớn.
- Chạm vào thông tin xác thực, cấu hình triển khai, hoặc hạ tầng không liên quan.

Không cần hỏi lại khi tác vụ đã rõ và rủi ro thấp. Thực hiện, kiểm chứng, rồi báo cáo.

## Kỳ vọng riêng của repository

Repository này phải dùng được cho AI agent khác. Vì vậy tài liệu phải rõ ràng, nhất quán, dùng tiếng Việt có dấu, và không phụ thuộc vào giả định cục bộ hoặc riêng tư.
